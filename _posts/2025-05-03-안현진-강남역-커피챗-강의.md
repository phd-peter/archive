## 배운 개발 용어들

### router
- 내가 어떤 인터넷 주소로 쳐고 들어갔을때. 어떤 행동이 나올지를 적어놓은 문서같은거.
- **참고:** [라우터란? (MDN)](https://developer.mozilla.org/ko/docs/Glossary/Router)
- **적용 예시:**
  - 웹 프레임워크(예: Express.js, Django)에서 URL별로 실행할 함수를 지정할 때 라우터를 사용함.

### 아키텍쳐
- 일종의 폴더 구조.
- 기능을 최대한 분리되어 놓음. 뭔가를 수정할때 최대한 분리되어있어서. 스스로 파악하기도 쉬운. (협업하기가 용이하고)
- 선배 개발자들이 만들어 놓은 유명 아키텍쳐가 많이 있다.
- 각 개발팀들은 그거를 가져다가 운영하는 서비스, 직원성향 등에 맞게 수정해서 사용하는거임
- **참고:** [소프트웨어 아키텍처란? (위키백과)](https://ko.wikipedia.org/wiki/%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4_%EC%95%84%ED%82%A4%ED%85%8D%EC%B2%98)
- **적용 예시:**
  - MVC(Model-View-Controller) 구조로 웹앱을 개발할 때, 각 역할별로 폴더를 분리함.

### 개발자
- 기본적으로 더 샤프한것을 하고싶어함. 그런 욕망들이 있는 존재임.
- 보통 프로젝트 리드가 먼저 아키텍쳐에 대한 큰 그림을 구상하고 의견을 수렴해서 결정한다. 2,3번 정도 의견교환후 아키텍쳐가 결정됨.
- 프로젝트 마무리까지 일반적으로는 그 결정된 아키텍쳐를 끌고간다.
- **참고:** [개발자란? (위키백과)](https://ko.wikipedia.org/wiki/%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4_%EA%B0%9C%EB%B0%9C%EC%9E%90)
- **적용 예시:**
  - 팀 프로젝트에서 아키텍처 설계 회의에 참여해 의견을 제시한 경험.

### Git
- 기본적으로 버전관리를 위한 툴이다.
    - 깃 클론해옴 - 연결은 있는 복사.
    - 깃 포크해옴 - 연결 끊어지는 느낌으로 복사해오기 - 별도의 추가적인 뭔가를 내 맘대로 할 수 있음.
    - 오픈소스 프로젝트에 기여한다. = 코드를 수정함 = pull request (관련 규칙이 어딘가에 적혀있음. master에 바로 하면안되고 어딘가에 하라고 정해져있음) 

- Git 관련 설명
    origin = repository 본체
    git push origin main : main 브랜치에 바로 push (현업에선 절대 없는 일)
    git push origin branch1 : branch1 이라는 브랜치에 push 
    git branch -> 빠져나오는 것은 q 누르면됨 : 전체 브랜치를 파악하는 command
    git checkout [브랜치이름] 으로 브랜치 바꿀수 있음 : command 창에서 브랜치 변경
    gpsup 명령어
- **참고:** [Git 공식 문서](https://git-scm.com/book/ko/v2)
- **적용 예시:**
  - 개인 프로젝트에서 새로운 기능을 개발할 때, feature 브랜치를 만들어 작업 후 pull request로 병합함.

### terminal
- 관련 명령어
    ls
    cd
    cd ..
- **참고:** [리눅스 기본 명령어 정리](https://www.lainyzine.com/ko/article/linux-basic-commands-summary/)
- **적용 예시:**
  - 프로젝트 폴더로 이동할 때 `cd ~/project` 명령어를 사용함.

### 기타 용어
- zsh [지셀] 이라고 읽음
- SDK = 밀키트 느낌. 여러 API를 하나하나 따와서 구현하기가 어려우니 개발사에서 직접 버무려 놓은 직통통로.
- **참고:** [zsh 공식 사이트](https://www.zsh.org/), [SDK란? (위키백과)](https://ko.wikipedia.org/wiki/%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4_%EA%B0%9C%EB%B0%9C_%ED%82%A4%ED%8A%B8)
- **적용 예시:**
  - macOS에서 기본 쉘을 zsh로 설정해 사용 중.
  - 카카오톡 오픈 API SDK를 활용해 챗봇을 개발한 경험.

### Cursor AI 단축키
- 커맨드 쉬프트 o -> 파일 찾기
- 커맨드 쉬프트 f -> 전체 단어 찾기 (파일 내부까지)
- **참고:** [Cursor 공식 문서](https://www.cursor.so/docs/shortcuts)
- **적용 예시:**
  - 여러 파일 중 원하는 파일을 빠르게 찾을 때 `Cmd+Shift+O` 단축키를 자주 사용함.
